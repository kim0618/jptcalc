# 애널리틱스 일일/주간 성과 리포트

## 목적

jptcalc, bumohyetaek, lovtaro 3개 사이트의 **GA4 + 서치콘솔 데이터**를 한 번에 조회해서 운영 인사이트 제공.

## 사이트별 오픈일 (컨텍스트)

| 사이트 | 오픈일 | 비고 |
|--------|--------|------|
| jptcalc | 2026-03-23 | 애드센스 4/28 재신청 예정 |
| bumohyetaek | 2026-04-03 | 정책/육아 정보 블로그 |
| lovtaro | 2026-04-06 | 인스타 타로 카드 |

**판단 기준:**
- 오픈 30일 미만 → 초기 단계, 색인/유입 적은 게 정상
- 오픈 30~90일 → 색인 본격화, 순위 변동 큼
- 오픈 90일 이상 → 안정화, 키워드 전략 유효

조언 시 오픈 일수를 고려해서 현실적 평가 제공 (예: 오픈 15일차인데 클릭 10회면 나쁘지 않음).

- 유입/조회수 변동 (전 기간 대비)
- 인기 페이지 / 인기 검색어
- **11~20위 키워드** 자동 추출 (보강 우선순위)
- **Thin Page 의심** 페이지 자동 탐지 (체류 15초 미만)
- 애드센스 재신청(4/28) 대비용 품질 점검

## 실행 방법

```bash
node /home/tjd618/.claude/analytics/report.mjs [period] [site]
```

### 파라미터

- `period` (선택, 기본값 `7d`):
  - `1d` — 어제 하루
  - `7d` — 최근 7일 (기본)
  - `28d` — 최근 28일 (서치콘솔 표준)
  - `30d` — 최근 30일
  - `90d` — 최근 90일
- `site` (선택, 기본 전체):
  - `jptcalc` / `bumohyetaek` / `lovtaro`

### 예시

```bash
# 전체 사이트 7일 리포트 (기본)
node /home/tjd618/.claude/analytics/report.mjs

# 어제 하루만
node /home/tjd618/.claude/analytics/report.mjs 1d

# jptcalc만 28일
node /home/tjd618/.claude/analytics/report.mjs 28d jptcalc

# lovtaro만 30일
node /home/tjd618/.claude/analytics/report.mjs 30d lovtaro
```

## 실행 지침

### 1. 사용자 요청 파싱

사용자 요청에서 기간과 사이트를 추출:

| 사용자 표현 | period |
|------------|--------|
| "어제", "오늘", "1일" | `1d` |
| "일주일", "7일", (생략) | `7d` |
| "4주", "28일" | `28d` |
| "한 달", "30일" | `30d` |
| "3개월", "90일" | `90d` |

| 사용자 표현 | site |
|------------|------|
| "jpt", "계산기" | `jptcalc` |
| "부모혜택", "bumo" | `bumohyetaek` |
| "로브타로", "타로", "인스타" | `lovtaro` |
| "전체", (생략) | (전체) |

### 2. Bash 도구로 스크립트 실행

해당 파라미터로 `node /home/tjd618/.claude/analytics/report.mjs [period] [site]` 실행.

### 3. 결과 해석 및 조언

스크립트 출력 그대로 사용자에게 전달하되, 다음 관점에서 **짧은 조언 2~3줄** 덧붙임:

- **애드센스 재신청(4/28) 관련**: Thin Page 많으면 `/bolster` 우선 작업 권고
- **11~20위 키워드**: `/blog` 또는 `/bolster`로 해당 키워드 보강 제안
- **체류시간 짧은 페이지**: 콘텐츠 품질 점검 필요
- **전 기간 대비 급감**: 원인 추정 (계절성/알고리즘/경쟁) 및 대응책

### 4. 저장 (선택)

사용자가 "저장해둬"라고 하면 `/home/tjd618/.claude/analytics/reports/YYYY-MM-DD.txt`에 리포트 저장.

## 주의사항

- **credentials.json은 절대 공유/커밋 금지** (이미 `.claude/analytics/`에 저장됨)
- API 할당량은 무료 티어 내에서 충분 (일일 수백 회 호출 가능)
- 데이터는 GA4/서치콘솔 처리 지연으로 **2~3일 전 데이터까지만 안정적**
- 너무 최근 데이터(오늘, 어제)는 부분 데이터일 수 있음
