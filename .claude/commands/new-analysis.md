## 콘텐츠 유형 (이 중에서만 작성)

### A. 비교 분석형 (comparison 템플릿)
- "A vs B" 형태의 두 제도/방법 비교
- 예: "기초연금 vs 국민연금 - 둘 다 받을 수 있을까?"
- 예: "요양원 vs 재가급여 - 부모님께 어떤 선택이 나을까?"
- 반드시 비교 기준표(list/info 섹션)와 상황별 추천을 포함

### B. 시뮬레이션형 (calculator 또는 standard 템플릿)
- 실제 금액/조건을 대입한 계산 시나리오
- 예: "은퇴 후 건강보험료, 월 얼마나 나올까? - 3가지 시나리오"
- 예: "기초연금 감액 없이 최대한 받는 방법 - 소득별 시뮬레이션"
- 반드시 구체적 금액, 조건, 결과를 포함

### C. 의사결정 가이드형 (checklist 템플릿)
- 상황별 판단 기준을 제시하는 가이드
- 예: "부모님 돌봄, 직접 vs 시설 - 결정 전 체크리스트"
- 예: "연금 조기수령 vs 연기수령 - 내 부모님은 어떤 게 유리할까?"
- 반드시 판단 기준(numbered-list)과 상황별 결론을 포함

### D. 종합 로드맵형 (standard 템플릿)
- 여러 기존 글을 엮는 시기별/단계별 종합 가이드
- 예: "60세 은퇴 후 첫 6개월 - 반드시 챙겨야 할 혜택 순서"
- 예: "부모님이 75세 넘었을 때 - 가족이 확인해야 할 5가지"
- 기존 글을 위키링크로 최소 3개 이상 연결

## 주제 선정 규칙

1. 기존 103개 글의 slug 전체를 먼저 확인
2. 기존 글과 70% 이상 겹치는 주제 금지
3. 기존 글 2~3개를 조합/비교하는 메타 콘텐츠 우선
4. 카테고리 균형 유지 (pension-welfare, health-care, finance-safety 골고루)
5. 선정한 주제를 먼저 보여주고 승인받은 후 작성

## 글 작성 규칙 (CONTENT-PROMPT.md 기반 + 추가)

1. CONTENT-PROMPT.md의 모든 필드 규칙, 품질 기준, A등급 체크리스트를 따른다
2. sections 5~8개, faq 2~3개, summary 80~150자
3. 위키링크 [[slug|텍스트]] 최소 3개 (기존 글 연결 강화)
4. 비교/시뮬레이션 관련 info 또는 tip 박스 최소 2개
5. 구체적 금액/조건/계산 결과 필수 포함
6. 작성 후 해당 카테고리의 index.ts에 import + 배열 추가
7. 관련 기존 글의 relatedSlugs에 새 글 slug 추가
8. npm run build로 빌드 확인
