# 제이퍼 정책 수치 최신화 스킬

## 목적

jptcalc.kr의 **세율·보험요율·기준값**을 정기 점검해 항상 최신으로 유지. 계산기 사이트는 수치 정확성 = 사용자 신뢰 = 검색 노출 직결. 블로그 글뿐 아니라 **계산기 페이지(/calc/**) HTML 안에 박힌 상수까지** 점검 대상.

## 마스터 데이터 파일

`/home/tjd618/jptcalc/data-points.md`

각 항목 구조:
- 항목명
- 현재 값
- 적용 시기 (예: 2026년)
- 공식 출처 URL
- 마지막 확인일
- 변동 주기 (연 1회 / 분기 / 한은 결정 시 / 법 개정 시 등)
- 검색 패턴 (영향 페이지 자동 식별용, 여러 표기 형태 포함)
- 영향 페이지 (블로그 + 계산기 경로 둘 다)

## 실행 원칙

- 사용자 명시 없으면 **5~7개 항목** 점검
- **마지막 확인일 오래된 순** + **변동 주기 도래 항목 우선**
- 공식 출처는 부모 Claude가 **직접 WebFetch**로 검증
- 변경 발견 시 **사용자 승인 후** 일괄 수정
- 마스터 파일이 비어 있거나 부족하면 첫 가동에서 시드 항목 채움

## 가동 절차

### Step 0: 마스터 파일 확인

`/home/tjd618/jptcalc/data-points.md` 존재 확인.
- 없으면: 시드 리스트로 골격 생성 후 사용자에게 보고
- 있으면: 영향 페이지 매핑 최신화 (각 검색 패턴을 blog/posts + calc/** 에 grep)

### Step 1: 점검 항목 선정

```
1. data-points.md 읽기
2. 모든 항목의 "마지막 확인일" 추출
3. 오래된 순으로 정렬
4. 변동 주기 도래 항목(예: 12~1월의 보험요율, 7월의 A값) 가중치 추가
5. 상위 5~7개 픽업 → 사용자에게 알림
```

### Step 2: 공식 출처 WebFetch 검증

각 항목에 대해:
1. `공식 출처` URL을 WebFetch
2. 페이지에서 현재 값 추출 시도
3. 추출 성공: 마스터 값과 비교
4. 추출 실패 (WAF·동적 페이지): 사용자에게 수동 확인 요청

### Step 3: 변경 발견 시 영향 페이지 식별

값이 변경된 항목에 대해:
1. `검색 패턴`을 `/home/tjd618/jptcalc/blog/posts/` 와 `/home/tjd618/jptcalc/calc/` 에 grep
2. 매칭된 파일 경로 목록 추출
3. 각 파일에서 매칭된 줄 번호와 컨텍스트 표시

### Step 4: 사용자 보고 + 승인 대기

```
## 변경 발견 항목

### 1. 2026년 최저임금
- 옛 값: 시급 9,860원 (2024년)
- 새 값: 시급 10,030원 (2026년 적용)
- 출처: 고용노동부 공고
- 영향 페이지: 8건
  - blog/posts/minimum-wage-2026.html (3곳)
  - blog/posts/hourly-wage-guide.html (2곳)
  - calc/salary/hourly-wage/index.html (상수 1곳)
  - ...

### 2. ...

이대로 일괄 수정할까요?
```

### Step 5: 일괄 수정 (승인 후)

각 영향 파일에 대해:
1. 매칭된 모든 위치 수정
2. 시점 표현 업데이트 ("2025년 기준" → "2026년 기준")
3. JSON-LD `dateModified` 필드 있으면 오늘 날짜로
4. 계산기 HTML 안의 JS 상수도 수정 (예: `const MIN_WAGE = 10030;`)
5. 누락 방지: 같은 파일에 옛 값이 남아있는지 재grep

### Step 6: 마스터 리스트 업데이트

`data-points.md`:
- 변경된 항목의 `현재 값` 갱신
- 모든 점검 항목의 `마지막 확인일` → 오늘
- 변경 이력 섹션 추가:

```markdown
### 2026-04-29
- 점검: 7개 (최저임금, 건강보험요율, ...)
- 변경: 2건 (최저임금 → 10,030원, 건강보험 → 7.15%)
- 영향 파일 수정: 14개
```

## 주의사항

- **수치 형식 다양성**: "10,030원" / "1만30원" / "시급 1만원" 등 표기 다름. 검색 패턴은 여러 형태 포함 필수
- **문맥 검증**: 같은 숫자가 다른 의미일 수 있음. 수정 전 컨텍스트 확인
- **계산기 JS 상수**: HTML 안 `<script>` 블록의 변수도 잊지 말 것 (ex: 누진공제 배열, 요율 상수)
- **공시지가·기준중위소득 같은 변동 큰 값**: 글에서 "2026년 기준"으로 명시되어 있어야 안전
- **카테고리 우선순위**: tax > finance > pension-welfare > salary > realestate > health/date/pet/ai
- **배포 금지**: 수정만 하고 rsync는 사용자가 직접 (사용자 룰)
- **불필요한 하이픈 리스트 금지**: 블로그 본문 수정 시 사용자 룰 준수
